# NR Fleet Manager

Centralized dashboard for managing multiple **Node-RED** instances.  
Monitor status, users, projects, and connect agents via WebSocket — all from one place.

---

## Features

- 🖥 **Instance monitoring** — real-time status, CPU, RAM, disk, uptime via [node-red-contrib-fleet-agent](https://www.npmjs.com/package/node-red-contrib-fleet-agent)
- 👥 **User management** — manage Node-RED users across instances
- 📁 **Project tracking** — Git branch, commit, dirty state per instance
- 🔑 **Universal Tokens** — connect agents without pre-registered instances
- 🏢 **Workspaces** — group instances and control access per team
- 🔐 **RBAC** — `super_admin` / `operator` / `viewer` roles with workspace-level access
- 📋 **Audit log** — full history of actions
- 🌗 **Dark mode** — built-in theme toggle

---

## Node-RED Agent

To collect metrics and manage your Node-RED instances, install the **Fleet Agent** on each Node-RED instance:

```bash
cd ~/.node-red
npm install node-red-contrib-fleet-agent
```

Then restart Node-RED, add a **Fleet Agent** node to your flow and configure:
- **Fleet URL** — `ws://your-fleet-server/ws`
- **Token** — Universal Token generated in the NR Fleet Manager dashboard

📦 **npm:** [node-red-contrib-fleet-agent](https://www.npmjs.com/package/node-red-contrib-fleet-agent)

## Quick Start (Docker)

**Option 1 — one-liner (downloads script, clones repo, then runs installer):**

```bash
curl -fsSL https://raw.githubusercontent.com/lotockii/nr-fleet-manager/main/install.sh | bash
```

This clones the repo into `./nr-fleet-manager` in the current directory, then runs the installer there.

**Option 2 — clone repo, then run installer:**

```bash
git clone https://github.com/lotockii/nr-fleet-manager.git
cd nr-fleet-manager
bash install.sh
```

The installer will:
1. Generate random secrets (`JWT_SECRET`, `DB_PASSWORD`, `ADMIN_PASSWORD`)
2. Optionally set up **HTTPS via Let's Encrypt** (if you have a domain)
3. Build and start all containers
4. Print the URL + credentials

> ⚠️ **Change the admin password on first login!**

## Architecture

```
┌─────────────┐     ┌──────────────┐     ┌──────────────────┐
│  Browser    │────▶│  nginx       │────▶│  Backend API     │
│  (React)    │     │  :80 / :443  │     │  Fastify :3001   │
└─────────────┘     └──────────────┘     └────────┬─────────┘
                                                   │
                                          ┌────────▼─────────┐
                                          │  PostgreSQL       │
                                          │  :5432            │
                                          └──────────────────┘
                                                   │ WebSocket
                                          ┌────────▼─────────┐
                                          │  Node-RED Agents  │
                                          │  (nr-fleet-agent) │
                                          └──────────────────┘
```

### Stack
| Layer     | Technology |
|-----------|-----------|
| Frontend  | React 18 + Vite + TailwindCSS |
| Backend   | Fastify + Drizzle ORM |
| Database  | PostgreSQL 16 |
| Auth      | JWT (HS256) |
| Transport | REST + WebSocket |

## Configuration (`.env`)

| Variable         | Description                        | Default         |
|------------------|------------------------------------|-----------------|
| `DB_NAME`        | PostgreSQL database name           | `nr_fleet`      |
| `DB_USER`        | PostgreSQL user                    | `nr_fleet`      |
| `DB_PASSWORD`    | PostgreSQL password (**required**) | —               |
| `JWT_SECRET`     | JWT signing secret (**required**)  | —               |
| `ADMIN_EMAIL`    | First admin email                  | `admin@localhost`|
| `ADMIN_PASSWORD` | First admin password               | —               |
| `PORT`           | HTTP port                          | `80`            |
| `ALLOWED_ORIGINS`| CORS origins (`*` for all)         | `*`             |
| `NODE_ENV`       | Environment                        | `production`    |

> Secrets are auto-generated by `install.sh`. Never commit `.env` to Git.

## Development

```bash
# Install dependencies
pnpm install

# Start backend (dev mode, hot-reload)
cd apps/workspace
pnpm dev

# Start frontend (dev mode, HMR)
cd apps/dashboard
pnpm dev

# Database migrations
cd apps/workspace
pnpm db:generate   # generate migration from schema changes
pnpm db:migrate    # apply migrations
```
